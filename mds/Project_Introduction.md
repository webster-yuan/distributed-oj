# 分布式 OJ 项目介绍

“简单介绍一下你的项目”时，你可以按照以下结构进行回答：**项目定位 -> 核心架构 -> 核心功能 -> 项目价值**。

---

## 1. 项目一句话简介
本项目是一个基于 **C++** 开发的**分布式在线判题系统 (Online Judge)**，旨在为用户提供代码在线编写、编译、运行及自动化测试的服务。

## 2. 项目背景
*   **学习动机**：为了深入理解 Linux 系统编程、网络编程以及分布式架构，我决定从零实现一个类似 LeetCode 或 Codeforces 的核心判题引擎。
*   **技术挑战**：传统的单机 OJ 在面对高并发请求时容易成为性能瓶颈，且运行用户提交的不可信代码存在极大的安全风险。

## 3. 核心架构设计
项目采用**微服务化**的思想，将系统拆分为两个核心模块：
*   **OJ Server (控制中心)**：基于 `httplib` 实现，负责题目管理、用户管理、负载均衡调度以及网页渲染（使用 `ctemplate`）。
*   **Compile Server (判题后端)**：负责核心的“编译+运行”业务。它是一个独立的服务，可以水平扩展部署在多台机器上。

## 4. 核心功能与技术点
*   **分布式调度**：OJ Server 维护一个后端服务池，通过**最小负载优先算法**将判题任务分发给最空闲的后端。
*   **安全沙箱**：在后端判题时，利用 `fork` 创建子进程，并通过 `setrlimit` 严格限制 CPU 和内存资源，防止恶意代码攻击。
*   **数据持久化**：使用 **MySQL** 存储题目数据、用户信息及答题进度。
*   **MVC 模式**：前端使用原生 HTML/JS，后端采用 Model-View-Controller 模式，实现逻辑与展示的分离。

## 5. 项目价值与成果
*   **高性能**：通过水平扩展判题后端，系统能够轻松应对高并发的代码提交请求。
*   **高可用**：实现了后端的**热插拔与自动容错**，单台判题服务器宕机不会影响整体服务的可用性。
*   **安全性**：构建了可靠的资源隔离机制，确保了服务器宿主机的运行安全。
